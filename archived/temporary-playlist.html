TEMPORARY PLAYLIST

A personal project, for fellow audiophiles.

BACKGROUND

Spotify is an audio streaming app that allows its users to create their own playlists, curating a collection of songs that they are enjoying at the moment. Similar to an artist’s album, these user-created playlists can be looped so that the user can enjoy their favourite songs over and over again.

EXPLORING THE PROBLEM

In order to create a playlist, the user has to go into the vertical ellipses beside the first song, select ADD TO PLAYLIST from the menu, select NEW PLAYLIST, and then give the playlist a name.

For each subsequent song, the user again has to go into the vertical ellipses beside the song, select ADD TO PLAYLIST from the menu, and select the newly created playlist.

In total, there are four steps to create the playlist, then three steps for every subsequent song.

Therefore, creating a playlist is a tedious process.

Being friends with a music aficionado, I know that there are particular days or moments that we want to listen to a selection of songs, but do not want to go through the process of creating a permanent playlist. Especially when a good playlist is a collection that takes time and careful consideration.

UNDERSTANDING THE USER

To understand the type of user that could benefit from addressing this problem, I created the persona below.

PROBLEM DEFINITION

Create a function on Spotify that allows the user to EFFICIENTLY create a PERSONALIZED collection of songs.

To EFFICIENTLY create a collection of songs would mean to do this in as few steps as possible.

To create a PERSONALIZED collection of songs would mean to give the user as much flexibility as they need. The user can add whatever songs they feel like adding at the time.

FUNCTIONALITY

I first needed to differentiate between functionalities associated with the existing state of the app and those associated with the new feature. They will be differentiated by the colour yellow.

Based on the flow, the first functionality to consider is creating and adding songs to the playlist.

The initial idea was to first enter a separate temporary playlist mode that prompted the user to select ALL the songs they wanted on the playlist before creating it.

However, the user should be able to go directly into the app with the intention of creating a temporary playlist and create it immediately. Making the user first enter the separate mode adds an unnecessary step. Additionally, this interaction could be difficult to use if the user wants to manually search for a particular song or add songs from several different playlists. They would have to keep track of the songs that they have selected while jumping from screen-to-screen.

I wanted to enable the user to add songs to the playlist efficiently, and as they wanted. The user taps on the plus sign next to the song to immediately create the temporary playlist and add the first song. Additional songs are added in the same way. The user can easily move around the app, adding songs as they please.

Once in the playlist, the functionality becomes similar to a typical Spotify playlist. The user can shuffle play and remove songs as necessary. However, with the goal of efficiency in mind, ensuring that song removal could be complete within one step was important. Therefore, the user easily removes a song by tapping on the X. Additionally, to emphasize the impermanence of this function, the user can also immediately clear the whole playlist with one tap.

REFLECTION

Mid-way into designing the function, I noticed that it was similar to Spotify’s already existing queue function. This function allows users to add songs to a queue that will play the songs in the order they are added. However, I believe that this temporary playlist solution is more accessible (the function is not hidden but instead visible throughout the app by its distinct colour) and flexible (it allows users to choose their own order and remove songs).
